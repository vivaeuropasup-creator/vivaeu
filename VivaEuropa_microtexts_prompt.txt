# PROMPT ДЛЯ CURSOR — внедрить «микротексты из жизни» по сайту

Ты — фронтенд-разработчик. Проект: статические HTML в папке `VivaEuropa/`, общий CSS. Задача — добавить короткие поясняющие микротексты на **русские страницы**. Вноси изменения **аккуратно и идемпотентно**: если блок уже добавлен, повторно не добавляй.

## 0) Глобальный CSS
В основном стилевом файле проекта (тот, где описаны базовые стили кнопок и сеток), **вставь единожды** следующие классы. Если такие классы уже существуют, не дублируй и не меняй их правила.

```css
/* Микротексты (заметки, списки, сноска) */
.note{
  font-size: 14.5px;
  line-height: 1.55;
  color:#8C8F98;              /* нейтрально-серый по гайдам */
  margin-top:8px;
}
.note-list{
  margin:10px 0 0;
  padding:0 0 0 18px;
  color:#8C8F98;
  font-size:14.5px;
  line-height:1.55;
}
.note-list li{ margin:4px 0; }

.footnote{
  margin-top:16px;
  font-size:13px;
  line-height:1.5;
  color:#8C8F98;
  opacity:.9;
}
```

Требования:
- Не менять базовую типографику страниц.
- Отступы вокруг новых элементов не должны ломать вертикальный ритм.

## 1) Главная: `VivaEuropa/index_ru.html`

### 1.1. Заметка под хиро-оффером
**Найди второй абзац хиро**, содержащий текст:
`Помогаем оформить шенгенскую визу с гарантированным результатом в визовом центре.`  
**Сразу после этого абзаца** вставь:

```html
<p class="note">Оплата только после одобрения TIE. Никаких предоплат за воздух.</p>
```

Убедись, что `<p class="note">` идет строго после второго параграфа оффера.

### 1.2. Микротексты в карточках цен

**Карточка «ВНЖ под ключ — 3500 €»**  
Найди карточку услуги с заголовком или контентом, содержащим `ВНЖ под ключ` и цену `3500`.  
Вставь **после краткого описания услуги** (если описания нет — **до блока цены**) следующий список:

```html
<ul class="note-list">
  <li>Готовим пакет, подаём, сопровождаем до сдачи отпечатков и выдачи TIE.</li>
  <li>Личный менеджер, без скриптов и навязчивых звонков.</li>
</ul>
```

**Карточка «Шенгенская виза — 400 €»**  
Найди карточку услуги с упоминанием `Шенгенская виза` и ценой `400`.  
Вставь аналогично (после текста карточки, до цены либо сразу под ценой — в зависимости от структуры) список:

```html
<ul class="note-list">
  <li>Пакет документов готовим мы, подача и биометрия в визовом центре — у вас.</li>
  <li>Сроки и маршрут подачи подбираем под ваш график.</li>
</ul>
```

Правило: если на карточке уже есть `.note-list` с идентичными пунктами — ничего не вставлять повторно.

## 2) Процесс: `VivaEuropa/process_ru.html`

После основного текста **каждого шага** добавь по одной заметке.

Найди блоки шагов по заголовкам:

- Шаг 1 заголовок содержит `Консультация` или `Консультация и анализ`.  
  **После основного абзаца шага** вставь:
  ```html
  <p class="note">Разбираем вашу ситуацию на фактах, а не по чек-листу из блога.</p>
  ```

- Шаг 2 заголовок содержит `Сбор документов` (или похожую формулировку).  
  Вставь после текста шага:
  ```html
  <p class="note">Делаем всё так, чтобы запросов от органов не прилетало, либо отвечаем на них за вас.</p>
  ```

- Шаг 3 заголовок содержит `Подача` (или `Подача и сопровождение`).  
  Вставь:
  ```html
  <p class="note">Сопровождаем до окна. Вы видите весь процесс в одном месте, без беготни.</p>
  ```

- Шаг 4 заголовок содержит `Получение` (или `Получение и поддержка`).  
  Вставь:
  ```html
  <p class="note">Запись на отпечатки включена. TIE забираете лично, мы подскажем как и где.</p>
  ```

**Юр-сноска в конце секции процесса**  
В конце основного раздела процесса (перед следующим крупным h2/h3 разделом, либо перед футером секции), вставь единожды:
```html
<p class="footnote">Решение выносит уполномоченная структура. Наша работа — подготовка и сопровождение до результата.</p>
```

Идемпотентность: если `.note`/`.footnote` уже вставлены, вторую копию не добавлять.

## Ограничения и что **не менять**
- Не менять размеры заголовков, цены, порядок блоков и кнопок.
- Не добавлять жирное начертание и не повышать размер шрифта у `.note`, `.note-list`, `.footnote`.
- Не вставлять микротексты на англоязычные страницы в рамках этого задания.
- Не плодить повторы: перед вставкой проверяй, что точный такой же блок рядом отсутствует.

## Проверка (Acceptance Criteria)
1. **CSS**: классы `.note`, `.note-list`, `.footnote` добавлены один раз глобально; не конфликтуют с существующими стилями, не ломают сетки.  
2. **Главная**:  
   - После второго абзаца оффера (про визы) есть `<p class="note">…TIE…</p>`.  
   - В карточке «ВНЖ под ключ — 3500 €» и в карточке «Шенгенская виза — 400 €» добавлены корректные `.note-list` с указанными пунктами.  
3. **Процесс**: у каждого из 4 шагов присутствует соответствующая `p.note`. В конце раздела — `p.footnote`.  
4. **Шенгенская виза (страница услуги)**: сразу под `<ol>` трёх шагов есть `ul.note-list` с двумя пунктами.  
5. **Услуги**: под вводным абзацем присутствует `ul.note-list` с двумя пунктами.  
6. **Города**: под блоком аренды каждого города присутствует `p.note` с фразой про ориентировочные цены; опциональная заметка про районы — только если есть соответствующий подблок.  
7. **Контакты**: под блоком кнопок есть `p.note` с режимом ответа и встречами в СПб.  
8. **FAQ**: перед футером есть `ul.note-list` с двумя «правда жизни» пунктами.  
9. **Идемпотентность**: повторного добавления блоков нет; исходные тексты не изменены.  
10. **Адаптив**: на мобильных новые тексты не “разъезжают” лейаут, отступы аккуратны, Lighthouse/axe не ругаются на доступность.

Сделай именно так. Если где-то структура отличается от описанной, используй ближайшие текстовые якоря (точные фразы выше) и вставляй в логически эквивалентное место, сохраняя внешний вид и ритм страницы.
